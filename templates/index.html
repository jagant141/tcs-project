    How It Works

    End-to-end ML pipeline — from raw data to live recommendations

    1
    Data Collection & Loading
    The TMDB 5000 Movies dataset is loaded using Pandas. It contains 5,000 films with metadata including title, genres, keywords, cast, crew, and a plot overview for each.
    pandas.read_csv()
    2
    Data Preprocessing
    JSON-encoded columns (genres, keywords) are parsed into plain text. Null values are filled. Genres, keywords and overview are concatenated into a single "combined features" document per movie.
    ast.literal_eval · fillna · str concat
    3
    TF-IDF Vectorization
    Each movie's combined text is converted into a numerical vector using Term Frequency–Inverse Document Frequency (TF-IDF). Rare, discriminative words get higher weights; common stop-words are ignored. Vocabulary is capped at 15,000 features with unigrams + bigrams.
    TfidfVectorizer(max_features=15000, ngram_range=(1,2))
    4
    Cosine Similarity Matrix
    A 5000 × 5000 pairwise similarity matrix is computed once at startup. Each cell (i, j) holds the cosine of the angle between movie i's and movie j's TF-IDF vectors — values closer to 1.0 mean more similar content.
    cosine_similarity(tfidf_matrix, tfidf_matrix)
    5
    Query & Rank
    When a user enters a title, its row in the similarity matrix is retrieved, sorted descending, and the top-N neighbours (excluding itself) are returned — each with a match percentage, rating, genre tags, and a short synopsis.
    sorted(sim_scores, reverse=True)[1:N+1]
    6
    Deployment via Flask
    The model is served as a REST API with Flask. The frontend calls POST /recommend and renders results dynamically — no page reload needed. An autocomplete endpoint (GET /search) enables live title suggestions.
    Flask · REST API · JSON · Vanilla JS
